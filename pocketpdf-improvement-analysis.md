## 二、UI 评估 (6 维度评分)

### 综合评分: 7.2 / 10 🟡

| 维度 | 分数 | 权重得分 | 评价 |
|------|------|----------|------|
| 一致性 Consistency | 7/10 | 1.75 (×25%) | Compose+XML 品牌色统一，但两套 UI 体系风格有差异 |
| 排版 Typography | 7/10 | 1.05 (×15%) | Material 3 标准字体系统，但缺少 CJK 字体定制 |
| 间距布局 Spacing | 7/10 | 1.40 (×20%) | 整体 8dp 网格感好，但 Reader 区域可优化 |
| 色彩 Color | 8/10 | 1.20 (×15%) | 紫色品牌色统一，暗色模式好，聊天气泡配色有改进空间 |
| 导航 Navigation | 6/10 | 0.90 (×15%) | 缺少底部导航、手势不支持自定义 |
| 简洁 Simplicity | 7/10 | 0.70 (×10%) | 整体清爽，但 Reader 控件占用空间偏大 |

### 逐屏分析

#### 1. ChatScreen (Compose) — 7.5/10 📱
✅ 好：
- 气泡颜色正确（用户=primary 底 + onPrimary 文字，AI=surfaceVariant 底）
- 流式输出体验好
- 页码引用可点击跳转
- 拖动自动滚到底部
- 长按菜单有复制/重新生成

⚠️ 问题：
- 气泡宽度上限 340dp 在平板/折叠屏上太窄
- 没有对话历史列表（只能逐文档对话）
- 没有消息删除/编辑
- 输入框没有@提及文档片段的功能

#### 2. ReaderScreen (XML View + Compose 混搭) — 7.0/10 📄
✅ 好：
- PdfPageView 自定义触控（双击缩放、双指缩放、平移）
- 淡入切换动画不错
- 页面覆盖滑块（Google Play Books 风格）
- 摘要按钮组 + BottomSheet 流式输出

⚠️ 问题：
- 底部翻页控件占用 64dp + 摘要栏 48dp = 112dp 阅读区
- 无法在 PDF 上做高亮/标注
- 没有文本搜索（Ctrl+F）
- 没有书签功能
- 翻页只有按钮，没有滑动手势翻页
- reader_surface 背景色是 surfaceVariant，但 reader 实际展示白色 PDF，中间有过渡不匹配

#### 3. LibraryScreen (XML RecyclerView) — 7.5/10 📚
✅ 好：
- MaterialCardView + 圆角 + 阴影
- 索引状态徽章色彩编码（灰/橙/绿/红）
- 左滑删除 + UNDO（5 秒倒计时）
- 空状态友好
- DiffUtil 增量更新

⚠️ 问题：
- 没有搜索/过滤栏
- 没有排序选项（只有时间倒序）
- 导入时没有进度百分比
- 没有文件夹/标签分类
- 没有批量操作（批量删除）

#### 4. SettingsScreen (XML) — 7.0/10 ⚙️
✅ 好：
- 模型预设切换（DeepSeek/通义千问/LM Studio）
- 测试连接 → 自动弹出模型下拉
- 密码切换可见
- 提示词计数器

⚠️ 问题：
- API Key 明文保存（需要加密存储）
- 没有导入/导出设置
- 分块策略只在设置里，应该在导入时也可选
- 缺少 Provider 支持（OpenAI、Claude、Groq 等）

#### 5. PingScreen (XML) — 7.5/10 📶
✅ 好：
- 简单自检界面
- Mono 字体展示结果

## 四、UI 界面改进

### 4.1 现有界面调整

**Reader 阅读器优化：**
1. **合并控件栏** — 将 `reader_controls`（64dp）和 `reader_summary_bar`（48dp）合并为底部浮动栏，减少垂直空间占用
2. **滑动手势翻页** — 在 PdfPageView 添加水平滑动翻页（缩放到 1x 时检测）
3. **沉浸模式** — 点击隐藏 Toolbar + 底部栏，全屏阅读
4. **页面进度条** — 顶部分段进度条（类似 Google Play Books），显示当前阅读位置

**Library 图书馆优化：**
1. 顶部搜索栏（SearchView）
2. 排序/过滤 ChipGroup
3. 拖动手感：拖动到文件夹的 Drag-and-Drop
4. 网格/列表视图切换

**Chat 聊天优化：**
1. 代码块语法高亮
2. 消息时间戳
3. 引用片段预览（引用某个 chunk 时可展开看原文）

### 4.2 设计建议

**全面拥抱 Material 3：**
- Chat 气泡改用完整 M3 配色（当前 `isDark -> Color.Black` 硬编码）
- Reader 工具栏背景色改用 `colorSurface` 而非硬编码
- 所有 Icon 使用 Material Icons Extended

**动画过渡：**
- Activity 之间使用共享元素过渡（Shared Element Transition）
- 导入 PDF 时卡片入场动画
- 翻页切换可加「书本翻页」效果（可选）

## 五、AI 能力增强

### 5.1 当前 AI 架构
```
User Question → RetrieveChunks (embedding similarity) → Build Prompt → LLM Stream → Display
                                                           ↑
                                                    PromptTemplates.ragQuery()
```

### 5.2 改进方向

| 改进 | 说明 | 复杂度 |
|------|------|--------|
| **多 Agent 架构** | 1) 查询分解 agent → 2) 多路检索 agent → 3) 综合回答 agent | 高 |
| **HyDE 检索** | 先用 LLM 生成假设答案，再用假设答案检索，提升召回率 | 中 |
| **重排序 (Rerank)** | 检索出 TOP K 后用 cross-encoder 重排序 | 中 |
| **图片理解** | PDF 中的图表/图片 → OCR/多模态理解 | 高 |
| **语音输入** | 语音提问（集成 Android SpeechRecognizer） | 低 |
| **Prompt 模板库** | 预设不同问答风格（简洁/详细/学术/口语） | 低 |
| **上下文记忆** | 多轮对话中的上下文窗口管理 | 中 |
| **流式引用** | 回答生成过程中逐步显示引用页码（目前是完整的） | 低 |

### 5.3 离线能力

你的 MediaPipe/ONNX 嵌入引擎已经是亮点。补充：
- **离线摘要**：小模型（Gemma 2B / Phi-3-mini）在端侧运行摘要
- **离线 QA**：用 ML Kit 或 MediaPipe 加载端侧 LLM 做基础问答

## 六、技术基建改进

| 领域 | 建议 | 说明 |
|------|------|------|
| **安全** | API Key 加密存储 | 当前 Preferences 明文存 `apiKey`，需用 EncryptedSharedPreferences 或 Android Keystore |
| **性能** | 延迟加载 Chunks | 索引文档时如果 PDF 超大（>100页），考虑分批索引 |
| **性能** | 图片缓存 | PDF 渲染使用 LRU 缓存已渲染的页面 Bitmap |
| **稳定性** | 崩溃日志 | 接入 Crashlytics / Sentry |
| **测试** | UI 自动化测试 | Compose UI Test + Espresso |
| **架构** | Activity → 单 Activity | 全部迁移到 Compose + Navigation Compose，移除 XML View 混用 |
| **CI** | GitHub Actions | 自动构建 + 单元测试 + Lint |

## 八、与竞品差异化建议

利用你的 AI 优势，做竞品做不了的事：

1. **「对话式阅读」** — 阅读 PDF 时随时调起 AI 浮窗问"这段什么意思"
2. **「智能批注」** — AI 自动提取关键段落并标注（类似 Kindle X-Ray）
3. **「跨文档知识库」** — 导入多个 PDF 后，对整库提问
4. **「PDF 地图」** — 用 AI 生成文档思维导图/大纲
5. **「语音问答」** — 开车时用语音提问，AI 朗读回答
6. **「学术模式」** — 专门优化论文阅读：公式识别、参考文献链式查询
